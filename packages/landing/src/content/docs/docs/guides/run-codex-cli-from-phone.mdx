---
title: How to run Codex CLI from your phone
description: Set up Zedra to control Codex CLI from iPhone, iPad, or Android while Codex keeps running on your own machine.
---

Codex CLI runs on your computer. Zedra connects your phone to that same workspace so you can answer approvals, steer the task, inspect files, and review diffs without moving your repository into a cloud IDE.

## What you need

- Codex CLI installed and signed in on your computer.
- Zedra installed on your computer and phone.
- A project directory you want to control from your phone.

## 1. Install Zedra on your computer

```sh
curl -fsSL zedra.dev/install.sh | sh
```

On Windows:

```powershell
powershell -c "irm zedra.dev/install.ps1 | iex"
```

## 2. Install the Codex integration

Run this once on the computer where Codex is installed:

```sh
zedra setup codex
```

This installs the hooks Zedra uses for session tracking and waiting notifications.

## 3. Start a Zedra workspace

Open the project directory you want to reach from your phone:

```sh
cd /path/to/project
zedra start
```

`zedra start` prints a QR code. Leave this process running while you pair the phone.

For a background daemon:

```sh
zedra start --detach
```

## 4. Pair your phone

1. Install Zedra from the [App Store](https://apps.apple.com/app/zedra-code-from-anywhere/id6760534630) or [Google Play](https://play.google.com/store/apps/details?id=dev.zedra.app).
2. Open the app.
3. Tap **Scan QR**.
4. Scan the QR printed by `zedra start`.

Pairing is one-time per workspace. The QR registers device keys; it does not move your source code into Zedra's cloud.

## 5. Start Codex from your phone

After pairing, open the workspace in Zedra and use the terminal to run Codex as usual. You can also start through the Zedra Codex skill:

```sh
$zedra-start
```

The phone terminal is interactive, so Codex approval prompts, command output, and follow-up instructions are visible from the phone.

## What Zedra adds around Codex

- Approve Codex commands from your phone.
- Read changed files in a mobile code editor.
- Review staged and unstaged git diffs.
- Read plans, specs, and reports in markdown preview.
- Get notified when Codex finishes or waits for approval.
