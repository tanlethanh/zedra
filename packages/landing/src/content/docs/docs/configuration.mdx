title: Configuration
description: Set daemon defaults once in a YAML config file — relays, telemetry, shell, agent session limits, and per-agent launch commands.
---

Set daemon defaults in a config file instead of passing flags on every `zedra start`. The file is optional; without it, Zedra uses its built-in defaults.

Create `~/.config/zedra/config.yaml` and add the settings you want:

```yaml
# ~/.config/zedra/config.yaml
relay_url:
  - https://sg1.relay.zedra.dev
no_telemetry: true
shell: /bin/zsh
agent_session_limit: 12
agents:
  hermes:
    launch_cmd: hermes --tui
  claude:
    bin: /opt/homebrew/bin/claude
```

Restart the daemon to apply changes:

```sh
zedra stop
zedra start
```

The daemon reads the file at startup. If the YAML is malformed, Zedra logs a warning and falls back to defaults, so a typo never blocks startup.

## File locations

Zedra reads config from two files:

- **Global** — `~/.config/zedra/config.yaml` (`%APPDATA%\zedra\config.yaml` on Windows). Defaults for every workspace.
- **Per-workspace** — `.zedra/config.yaml` in the workspace directory you serve. Settings here apply only to that workspace and override the global file. Commit it to share project defaults with your team.

Both files use the same settings. Set a value in one or both.

## Precedence

Settings resolve from lowest to highest priority:

1. Built-in defaults
2. Global `~/.config/zedra/config.yaml`
3. Per-workspace `.zedra/config.yaml`
4. Environment variables
5. `zedra start` flags

A flag or environment variable always overrides both files. Each file fills in what the level below it did not set.

## Settings

### `relay_url`

Relay servers the daemon advertises for connections. Defaults to Zedra's built-in relays. A `zedra start --relay-url` flag overrides it.

### `no_telemetry`

Set `true` to disable anonymous telemetry for every workspace. This matches `zedra start --no-telemetry` and `ZEDRA_TELEMETRY=0`. See [Telemetry](/docs/telemetry) for what Zedra collects.

### `shell`

Path to the shell that terminals launch. It overrides the inherited `$SHELL`, so phone terminals use the shell you name regardless of how the daemon started.

### `agent_session_limit`

Maximum concurrent managed-agent sessions, clamped to a safe range. The `ZEDRA_AGENT_SESSION_LIMIT` environment variable overrides it.

### `agents`

Per-agent overrides, keyed by the agent's registry slug (`claude`, `codex`, `opencode`, `hermes`, `pi`, `maki`). Each entry takes:

- `bin` — the executable to launch instead of the one on your `PATH`. Use it when the agent CLI lives at a non-standard path.
- `launch_cmd` — the full command run when starting the agent. It overrides `bin`. Use it to add flags, such as launching Hermes in its TUI with `hermes --tui`.

Agents you do not list keep their default launch command.
